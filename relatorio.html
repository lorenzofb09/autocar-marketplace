1. Introdução

Este relatório apresenta o desenvolvimento do projeto AutoCar Marketplace, uma plataforma web moderna para compra e venda de veículos. O projeto foi desenvolvido como trabalho prático da disciplina de Desenvolvimento Front-End no 1º período do curso de Sistemas de Informação pela UNESC.

O AutoCar Marketplace foi concebido com o objetivo de aplicar os conhecimentos adquiridos em HTML5, CSS3 e JavaScript, criando uma aplicação web funcional, acessível e responsiva que simula um ambiente real de e-commerce automotivo.

Durante o desenvolvimento, foram implementadas diversas funcionalidades como busca de veículos em tempo real, sistema de favoritos, galeria de imagens interativa, compartilhamento em redes sociais e sistema de notificações visual, tudo utilizando tecnologias modernas do desenvolvimento web.

Objetivo Geral: Desenvolver uma aplicação web completa que demonstre domínio das tecnologias front-end fundamentais, com foco em qualidade de código, acessibilidade e experiência do usuário.

2. Objetivos

2.1 Objetivos Gerais

  • Desenvolver uma aplicação web funcional e atraente para um marketplace de veículos
  • Aplicar conhecimentos de HTML5, CSS3 e JavaScript de forma prática
  • Demonstrar capacidade de criar interfaces modernas e responsivas
  • Implementar boas práticas de desenvolvimento web

2.2 Objetivos Específicos

  • Criar interface intuitiva e agradável para visualização de veículos
  • Implementar sistema de busca e filtros funcionais
  • Desenvolver sistema de favoritos com persistência em localStorage
  • Criar galeria de imagens interativa com navegação por teclado e mouse
  • Implementar compartilhamento em redes sociais (WhatsApp, Facebook, Twitter, Email)
  • Garantir acessibilidade (WCAG 2.1 nível AA)
  • Tornar a aplicação responsiva para dispositivos móveis, tablets e desktops
  • Implementar sistema de notificações visuais
  • Utilizar apenas Vanilla JavaScript, sem dependências externas
Resultado Esperado: Uma aplicação web completa, funcional e pronta para apresentação, demonstrando competências fundamentais em desenvolvimento front-end.

3. Descrição do Projeto

3.1 Visão Geral

O AutoCar Marketplace é uma plataforma de comércio eletrônico especializada em veículos. A aplicação permite que usuários visualizem uma lista de veículos disponíveis para venda, acessem detalhes específicos de cada anúncio, salvem seus veículos favoritos e compartilhem anúncios em redes sociais.

A plataforma foi desenvolvida com foco em experiência do usuário, garantindo navegação intuitiva, carregamento rápido de imagens e compatibilidade com diversos dispositivos. Todos os dados são gerenciados no navegador do cliente utilizando localStorage, permitindo uso sem necessidade de backend neste estágio de desenvolvimento.

3.2 Público-alvo

  • Compradores: Pessoas interessadas em comprar veículos usados
  • Vendedores: Proprietários de veículos que desejam vender
  • Entusiastas de Carros: Pessoas que acompanham o mercado automotivo
  • Usuários em Geral: Qualquer pessoa com acesso à internet via web browser

3.3 Estrutura de Arquivos

O projeto segue uma organização clara e profissional:

autocar-marketplace/ ├── index.html (Página inicial) ├── detalhes.html (Página de detalhes do veículo) ├── relatorio.html (Este arquivo) ├── css/ │ ├── style.css (Estilos página inicial) │ ├── detalhes.css (Estilos página detalhes) │ └── normalize.css (Normalização CSS) ├── js/ │ ├── script.js (Lógica página inicial) │ ├── detalhes.js (Lógica página detalhes) │ └── galeria.js (Galeria de imagens) └── README.md (Documentação do projeto)

4. Funcionalidades Implementadas

4.1 Página Inicial (index.html)

4.1.1 Header e Navegação

  • Logo da marca "🚗 AutoCar Marketplace"
  • Barra de busca em tempo real com validação
  • Menu de navegação com links: Sobre, Contato, Minhas Compras
  • Navegação acessível com atributos ARIA

4.1.2 Grid de Veículos

  • Exibição de veículos em layout responsivo com CSS Grid
  • Cards com imagens, preço, especificações e ações
  • Badge "Destaque" para anúncios promocionais
  • Informações: Transmissão, Quilometragem, Combustível, Localização

4.1.3 Sistema de Favoritos

  • Botão para adicionar/remover favoritos (❤️)
  • Persistência de favoritos no localStorage
  • Indicador visual de favorito ativo
  • Sincronização entre abas do navegador

4.1.4 Sistema de Busca

  • Busca em tempo real com debounce de 300ms
  • Filtro por marca, modelo e preço
  • Mensagem "Nenhum resultado encontrado"
  • Busca case-insensitive

4.2 Página de Detalhes (detalhes.html)

4.2.1 Visualização do Veículo

  • Imagem grande do veículo com carregamento lazy
  • Título e preço em destaque
  • Especificações completas do veículo
  • Botão "Voltar" para retornar à página inicial

4.2.2 Funcionalidade de Compra

  • Botão "Comprar Agora" com modal de confirmação
  • Validação de dados antes da compra
  • Notificação de sucesso após confirmação
  • Simulação de processo de compra

4.2.3 Compartilhamento em Redes Sociais

  • Botão "SHARE" para compartilhamento
  • Integração com: WhatsApp, Facebook, Twitter, Email
  • Mensagens personalizadas com dados do veículo
  • URLs dinâmicas com parâmetros de rastreamento

4.3 Galeria de Imagens Interativa

  • Navegação por clique em botões
  • Navegação por teclado (setas esquerda/direita)
  • Thumbnails para visualização rápida
  • Contador de imagens (ex: "3 de 5")
  • Suporte a teclado para acessibilidade

4.4 Sistema de Notificações

  • Notificações visuais com animação
  • Tipos: Sucesso, Erro, Aviso, Informação
  • Auto-fechamento após 3 segundos
  • Suavização de entrada e saída

6. Estrutura Técnica

6.1 HTML5 Semântico

O projeto utiliza elementos semânticos do HTML5 para melhor estrutura e significado:

  • <header> - Cabeçalho e navegação
  • <main> - Conteúdo principal
  • <section> - Seções temáticas
  • <article> - Artigos independentes (cards)
  • <nav> - Navegação
  • <figure> e <figcaption> - Imagens com legendas

Exemplo de estrutura semântica:

<article class="card"> <figure class="card-image-wrapper"> <img src="..." alt="Descrição"> </figure> <section class="card-info"> <h2 class="card-title">Título</h2> <!-- conteúdo --> </section> </article>

6.2 CSS3 Moderno

O projeto utiliza funcionalidades modernas do CSS3:

  • CSS Grid: Layout do grid de veículos
  • Flexbox: Alinhamento de componentes
  • Media Queries: Design responsivo
  • Animações: Transições suaves e keyframes
  • Custom Properties: Variáveis CSS (cores, espaçamentos)
  • Pseudo-elementos: Efeitos visuais sem elemento adicional

Exemplo de Grid responsivo:

.vehicles-grid { display: grid; grid-template-columns: repeat(auto-fill, minmax(300px, 1fr)); gap: 20px; padding: 20px; } @media (max-width: 768px) { .vehicles-grid { grid-template-columns: 1fr; } }

6.3 JavaScript ES6+

O código JavaScript utiliza recursos modernos:

  • Arrow Functions: Sintaxe concisa
  • Destructuring: Extração de valores
  • Template Literals: Strings dinâmicas
  • Async/Await: Operações assíncronas
  • Event Listeners: Manipulação de eventos
  • DOM API: Manipulação do DOM

Exemplo de função moderna:

const handleFavorite = (vehicleId) => { const favorites = JSON.parse( localStorage.getItem('favorites') || '[]' ); if (favorites.includes(vehicleId)) { favorites.splice( favorites.indexOf(vehicleId), 1 ); } else { favorites.push(vehicleId); } localStorage.setItem( 'favorites', JSON.stringify(favorites) ); };

7. Acessibilidade e Responsividade

7.1 Acessibilidade (WCAG 2.1)

O projeto segue as diretrizes de Acessibilidade para Conteúdo Web (WCAG 2.1) em nível AA:

7.1.1 Navegação por Teclado

  • Todos os botões e links são acessíveis via Tab
  • Ordem de tabulação lógica (left-to-right, top-to-bottom)
  • Enter para ativar botões e links
  • Setas para navegação em galerias
  • Escape para fechar modais

7.1.2 Leitores de Tela

  • Atributos aria-label em botões iconizados
  • Atributos aria-labelledby em seções
  • Atributos alt descritivos em imagens
  • Estrutura de heading hierárquica (h1, h2, h3)
  • Uso de <label> associadas aos inputs

Exemplo de elemento acessível:

<button class="btn btn-primary" onclick="handlePurchase()" aria-label="Comprar Honda Civic Touring 2022"> 🛒 Comprar Agora </button>

7.1.3 Contraste e Legibilidade

  • Razão de contraste mínima 4.5:1 para texto
  • Cores não como único meio de comunicação
  • Fonte legível (16px mínimo em mobile)
  • Espaçamento adequado entre elementos

7.2 Design Responsivo

A aplicação é totalmente responsiva em 3 principais breakpoints:

Dispositivo Largura Características
Mobile < 768px 1 coluna, layout empilhado, buttons fullwidth
Tablet 768px - 1023px 2 colunas, menu adaptado
Desktop ≥ 1024px 3-4 colunas, layout completo

7.2.1 Técnicas Responsivas Implementadas

  • viewport meta tag para escalabilidade
  • Media Queries para adaptar layout
  • Imagens responsivas com srcset
  • CSS Grid com auto-fill e minmax
  • Flexbox para componentes flexíveis
  • Font-size relativo (rem, em)

7.2.2 Testes de Responsividade

  • iPhone SE (375px)
  • iPad (768px)
  • Desktop 1920x1080
  • Testes em Chrome DevTools
